## Group-Specific Adaptations

After joining each group, check and document:

1. **Does the group allow links?** Some groups strip links. If so, put WhatsApp numbers only.
2. **Does the group require admin approval for posts?** If yes, allow 24-48h for approval.
3. **What's the posting culture?** Some groups are discussion-heavy (adapt to conversational format). Some are listing-heavy (use direct format).
4. **Are there explicit rules?** Many groups post rules as pinned. Read them. Violating rules = ban.
5. **How active is the group?** Groups with <100 members or <1 post/week may not be worth posting to.

Document all findings in the registry's `admin_posting_rules` field for each group.

---

## Tracking

After each group post:
1. Log in _completed/YYYY-MM-DD/ per existing workflow
2. Update the group's `last_posted` field in the registry
3. Track engagement (likes, comments, shares) in the next morning metrics check
4. If a post is deleted by admins, update `admin_posting_rules.notes` and reduce posting frequency

---

## Escalation

- If a group admin contacts you about posting: respond politely, offer to adjust frequency or content type
- If banned from a group: update `join_status` to "rejected", set `is_active_for_posting` to false
- If a post generates a HOT lead (asks about reservation, contract, pricing): follow the HOT LEAD protocol from CRM-LEAD-SCORING.md (contact within 2 hours)
